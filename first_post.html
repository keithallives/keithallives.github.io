Static & Dynamic Websites

February 09, 2019

You probably have 2 ways to make your website: static or dynamic. Let's try to break it down to help you decide which way to take.

Static Website

It is a very simple and basic website. It doesn't have much functionality to it. We use HTML, CSS and vanilla JS to build a static website.

Let's take a look at the Pros & Cons:

Pros:

  • fast — no lengthy load times when displaying the page
  • cheap—often cheaper to host static files on Netlify, S3, etc than a hosted environment
  • Lsimpler for a developer to setup

Cons:

  • updates-requires a developer to update content, not a nice CMS as provided by WordPress and others

Dynamic Website

This website is simply a static website with a twist. Meaning a lot of things is going on behind the scene. We use web framework such as Django(Python), Laravel(PHP) and Express(JS) to build a dynamic website.

So, Here are the Pros & Cons:

Pros:

  • dynamic content generated for each user
  • easy to update with a CMS like Wordpress
  • more powerful with features like login, payments, etc.

Cons:

  • expensive—setting up, maintaining, and scaling a database is costly
  • complex—much more steps involved to setup and deploy vs a static site

The Bottomline

If you need features like login, payments, etc. then it's going to be a dynamic website. But if you just need a simple website for blogging or for promoting your business online, you might consider setting up a static website.
